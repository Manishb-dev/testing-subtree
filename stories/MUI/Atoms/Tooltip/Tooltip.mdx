# Tooltip

A lightweight informative popup that appears when the user hovers, focuses, or taps a trigger element. Use tooltips to provide supplemental context for icon buttons, abbreviated labels, or form fields that need extra explanation.

---

## Usage

```tsx
import { Tooltip, Button } from '@bmi/tonic-components-mui';

<Tooltip title="Helpful information goes here" placement="top">
  <Button variant="outlined">Hover over me</Button>
</Tooltip>
```

## Placements

The `placement` prop controls which side of the trigger the tooltip appears on.
Tonic supports all four primary directions (`top`, `bottom`, `left`, `right`) as
well as their `*-start` and `*-end` sub-variants for edge-aligned positioning.

## With Arrow

Tonic defaults `arrow` to `true`, adding a small directional pointer that visually
connects the tooltip to its trigger. Disable it for floating-label style callouts.

## Under the Hood

This component wraps [`MuiTooltip`](https://mui.com/material-ui/api/tooltip/).

**MUI base:** `@mui/material/Tooltip`  
**Theme override:** `packages/theme/src/overrides/tooltip.ts`

All visual styling (background, typography, border-radius, arrow colour) lives in
the theme override. The component's only job is to enforce Tonic prop defaults and
forward everything else to MUI.

## Accessibility

- Always provide a meaningful `title` — screen readers announce it as the accessible name for icon-only triggers.
- Wrap non-interactive elements (e.g. disabled buttons, `<span>`) in a `<span>` so mouse events can propagate.
- Tooltips are not a substitute for persistent help text — use helper text or labels for critical information.
- The tooltip is announced via `aria-describedby` automatically by MUI.

---

## Do / Don't

**Do**
- Use tooltips to clarify icon-only buttons where the icon alone is ambiguous
- Keep tooltip text concise — one short sentence or a few words
- Always wrap disabled MUI buttons in a `<span>` to ensure tooltip events fire

**Don't**
- Don't put interactive content (links, buttons) inside the tooltip `title`
- Don't use tooltips as the sole source of required information
- Don't rely on tooltip text for mobile-only flows — touch devices may not trigger hover
