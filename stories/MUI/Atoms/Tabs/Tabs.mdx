# Tabs

Navigation bar that groups related views behind pill-shaped tab items. Use Tabs to let users switch between peer-level content panels without leaving the page.

---

## Usage

```tsx
import { Tabs, Tab } from '@bmi/tonic-components-mui';

const [value, setValue] = useState(0);

<Tabs value={value} onChange={(_, v) => setValue(v)}>
  <Tab label="Dashboard" value={0} />
  <Tab label="Analytics" value={1} />
  <Tab label="Reports" value={2} />
</Tabs>
```

## States

Each `Tab` can be in one of three states: unselected (default), selected, or disabled.
The selected tab receives a solid `paper4` background fill with white text and a semi-bold weight.
The default tab shows secondary text with no background.

## Under the Hood

This component wraps [`MuiTabs`](https://mui.com/material-ui/api/tabs/) and [`MuiTab`](https://mui.com/material-ui/api/tab/).

**MUI base:** `@mui/material/Tabs` + `@mui/material/Tab`  
**Theme override:** `packages/theme/src/overrides/tabs.ts`

The Tonic Tabs design replaces MUI's default underline indicator with a pill-shaped background fill on the selected tab. The indicator is hidden via the theme and a `backgroundColor` + `borderRadius` override is applied to `.Mui-selected` on the `MuiTab` slot.

## Accessibility

- `Tabs` renders a `role="tablist"` container; each `Tab` is a `role="tab"` button. The associated panel must have `role="tabpanel"` and be linked via `aria-controls` / `aria-labelledby`.
- Pass `aria-label` or `aria-labelledby` to the `Tabs` component to identify the navigation region for screen readers.
- Keyboard navigation: `←` / `→` (or `↑` / `↓` in vertical mode) moves focus between tabs. `Enter` or `Space` activates the focused tab.
- Disabled tabs remain focusable by keyboard but emit no `onChange` event and convey their state via `aria-disabled="true"`.

## Do / Don't

**Do**
- Keep tab labels short — one to two words that clearly describe the content panel.
- Use `Tabs` for peer-level content switching, not for multi-step flows (use `Stepper` instead).
- Always bind `value` and `onChange` — uncontrolled Tabs are not supported by this component.
- When using `fullWidth`, limit tabs to 2–4 items so each label stays readable.

**Don't**
- Don't use Tabs as a page-level router without also updating the URL when the tab changes.
- Don't nest Tabs inside another Tabs instance — use a single navigation hierarchy.
- Don't mix disabled and active tabs unless there is a clear reason (e.g. permissions-based access).
