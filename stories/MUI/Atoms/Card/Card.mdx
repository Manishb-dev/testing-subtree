# Card

A themed surface container for grouping related content and actions. Wraps MUI Card with the Tonic design vocabulary — no shadow by default, zero-padding content and action areas that consumers control via `sx` or inner layout.

---

## Usage

```tsx
import { Card, CardContent, CardActions, Button, Typography } from '@bmi/tonic-components-mui';

<Card variant="outlined" sx={{ maxWidth: 345 }}>
  <CardContent sx={{ p: 2 }}>
    <Typography variant="h6">Title</Typography>
    <Typography variant="body2" color="text.secondary">
      Supporting text content.
    </Typography>
  </CardContent>
  <CardActions sx={{ p: 2, pt: 0 }}>
    <Button variant="text" size="small">Action</Button>
  </CardActions>
</Card>
```

## Under the Hood

This component wraps [`MuiCard`](https://mui.com/material-ui/api/card/).

**MUI base:** `@mui/material/Card`, `@mui/material/CardContent`, `@mui/material/CardActions`
**Component styles:** `packages/components/src/atoms/Card/Card.styles.ts`

All visual styling lives in the `styled()` wrappers. The component's only job is to enforce the Tonic prop vocabulary and forward everything to MUI.

## Accessibility

- Use semantic heading elements inside `CardContent` (e.g. `<Typography variant="h6">`) so screen readers can navigate card titles.
- When a card is interactive (clickable), add `role="button"` and `tabIndex={0}` or wrap in a link.
- Group related actions in `CardActions` so they form a logical unit for keyboard navigation.

---

## Do / Don't

**Do**
- Use `CardContent` and `CardActions` as direct children of `Card` for proper structure
- Control spacing via `sx={{ p: 2 }}` on `CardContent` / `CardActions` rather than on the Card itself
- Use `variant="outlined"` when cards are placed on coloured or busy backgrounds for visual separation

**Don't**
- Don't nest interactive cards inside other interactive elements (buttons, links)
- Don't use the elevation variant expecting a shadow — Tonic removes the default MUI box-shadow
