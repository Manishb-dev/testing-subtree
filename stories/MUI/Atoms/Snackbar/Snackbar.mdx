# Snackbar

Temporary feedback notification for time-sensitive user actions such as license renewals, expirations, and suspensions.

---

## Usage

```tsx
import { Snackbar } from '@bmi/tonic-components-mui';

const [open, setOpen] = useState(false);

<Snackbar
  open={open}
  severity="success"
  message="License has been renewed successfully."
  autoHideDuration={4000}
  onClose={() => setOpen(false)}
/>
```

## Under the Hood

This component wraps [`MuiSnackbar`](https://mui.com/material-ui/api/snackbar/) and renders a [`MuiSnackbarContent`](https://mui.com/material-ui/api/snackbar-content/) internally.

**MUI base:** `@mui/material/Snackbar` + `@mui/material/SnackbarContent`
**Theme override:** `packages/theme/src/overrides/snackbar.ts`

The component composes severity-specific icons from `@bmi/tonic-icons-web` into the message slot. All background colors, typography, padding, and shadow styles live in the `MuiSnackbarContent` theme override. The component itself is purely structural.

## Accessibility

- The `SnackbarContent` renders with `role="alert"` by default, ensuring screen readers announce the message.
- When using `autoHideDuration`, keep it above 4000 ms so users have time to read the content.
- Always provide a close mechanism (action button or `onClose`) to allow keyboard users to dismiss the notification.
- Avoid stacking multiple snackbars simultaneously — use a queue pattern if multiple messages must be shown.

---

## Do / Don't

**Do**
- Use `success` severity to confirm completed user actions
- Use `warning` severity for time-sensitive notices that require attention
- Use `error` severity for failures that block the user's workflow
- Keep messages concise — one sentence maximum

**Don't**
- Don't use Snackbar for persistent information — use Alert instead
- Don't set `autoHideDuration` below 4000 ms for messages the user needs to read
- Don't stack multiple snackbars on screen at the same time
- Don't include complex interactive content inside the message — keep it to plain text
