# Avatar

A Tonic wrapper around [MUI Avatar](https://mui.com/material-ui/react-avatar/) that enforces the Tonic size vocabulary and supports three content types: text initials, icons, and images.

---

## Usage

```tsx
import { Avatar } from '@bmi/tonic-components-mui';
import { StarIcon } from '@bmi/tonic-icons-web';

{/* Text initials */}
<Avatar size="extraLarge">AS</Avatar>

{/* Icon content */}
<Avatar size="large">
  <StarIcon size={20} />
</Avatar>

{/* Image */}
<Avatar size="extraLarge" src="/path/to/photo.jpg" alt="User name" />
```

## Content Types

Avatar supports three content modes determined by the props you pass:

- **Text** - pass a short string (typically 1–2 initials) as `children`
- **Icon** - pass a Tonic icon component as `children`
- **Image** - set the `src` and `alt` props for a photo avatar

## Under the Hood

This component is a thin wrapper around [`MuiAvatar`](https://mui.com/material-ui/api/avatar/).

**MUI base:** `@mui/material/Avatar`
**Theme override:** `packages/theme/src/overrides/avatar.ts`

All visual styling - sizes, padding, font sizes, icon sizing, background color - lives in the theme override. The component's only job is to enforce the Tonic size vocabulary and forward everything to MUI.

## Accessibility

- Always provide `alt` text when using image avatars so screen readers can identify the person.
- For text/icon avatars used as decorative elements, pass `aria-hidden="true"` if the name is conveyed by adjacent text.
- Use meaningful initials (e.g. first + last name) rather than arbitrary characters.

---

## Do / Don't

**Do**
- Use text avatars with 1–2 character initials for the best visual fit
- Pair image avatars with an `alt` prop describing the person
- Use the `size` prop rather than overriding dimensions via `sx`

**Don't**
- Don't use more than 2 characters as text content - it overflows smaller sizes
- Don't hardcode pixel dimensions with `sx` - use the `size` prop instead
- Don't omit `alt` on image avatars - it breaks accessibility
