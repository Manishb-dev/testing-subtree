# Dialog

Centered popup dialog for confirmations, status notifications, and simple user decisions. Renders an optional status icon, heading, body text, and one or two action buttons over a backdrop.

---

## Usage

```tsx
import { Dialog } from '@bmi/tonic-components-mui';
import { CheckCircleIcon } from '@bmi/tonic-icons-web';

<Dialog
  open={isOpen}
  onClose={() => setOpen(false)}
  icon={<CheckCircleIcon size="large" variant="filled" sx={{ color: 'success.main', width: 82, height: 82, fontSize: 82 }} />}
  heading="Payment Successful"
  body="Your payment has been processed successfully."
  primaryActionLabel="Done"
  onPrimaryAction={() => setOpen(false)}
/>
```

## Under the Hood

This is a **shared** component composed from base Tonic primitives.

**Uses:** `Dialog` (MUI), `Button` (Tonic), `Typography` (Tonic), `Stack` (Tonic)
**Styled wrapper:** `packages/components/src/shared/Dialog/Dialog.styles.ts`

The dialog's paper surface is styled to match the Tonic popup card design — white background, light border, 8px radius, fixed 348px width.

## Accessibility

- The dialog traps focus automatically via MUI's built-in focus management.
- Pressing Escape fires `onClose` by default.
- The `heading` text serves as the accessible dialog title. For screen readers, consider passing `DialogProps={{ 'aria-labelledby': 'dialog-title' }}` when custom heading markup is needed.

---

## Do / Don't

**Do**
- Use filled icon variants at `large` size with custom `sx` dimensions and semantic color tokens for clear status communication
- Keep heading and body text concise — this is a confirmation dialog, not a page
- Always provide `onClose` so users can dismiss via backdrop or Escape

**Don't**
- Don't nest forms or complex layouts inside the dialog body — use a full page or drawer instead
- Don't use more than two action buttons — the dialog is designed for one primary and one optional secondary action
- Don't hardcode hex colors on icons — use MUI palette tokens like `success.main` or `error.main`
