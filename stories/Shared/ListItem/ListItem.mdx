# ListItem

Interactive list entry for navigation menus, selection lists, and option panels. Supports primary text, secondary text, avatars, end icons, and checkboxes through composable props.

---

## Usage

```tsx
import { ListItem } from '@bmi/mui-tonic-components';

<ListItem primaryText="Account Settings" onClick={() => navigate('/settings')} />
```

## Under the Hood

This component composes [`MuiListItemButton`](https://mui.com/material-ui/api/list-item-button/), [`MuiListItemText`](https://mui.com/material-ui/api/list-item-text/), [`MuiListItemAvatar`](https://mui.com/material-ui/api/list-item-avatar/), and [`MuiListItemIcon`](https://mui.com/material-ui/api/list-item-icon/).

**MUI base:** `@mui/material/ListItemButton` + layout primitives
**Theme override:** `packages/theme/src/overrides/list.ts`

All visual styling lives in the theme override. The component's job is to orchestrate the correct MUI sub-components based on the provided props.

## Accessibility

- `ListItemButton` renders as a `role="button"` element, accessible via keyboard (Enter/Space).
- When `showCheckbox` is true, the checkbox is independently focusable with its own ARIA checked state.
- Provide meaningful `primaryText` values — screen readers announce the item's text content when navigating the list.
- Wrap list items in a `<List>` component to ensure proper `role="list"` semantics.

---

## Do / Don't

**Do**
- Use `selected` to indicate the currently active item in a navigation or selection list
- Use `secondaryText` for supplementary details like descriptions, dates, or status labels
- Wrap items in a `<List>` for proper semantic structure

**Don't**
- Don't use `showCheckbox` and `avatar` together — pick one leading element per item
- Don't embed complex interactive content inside `primaryText` or `secondaryText` — keep them as plain text or simple nodes
- Don't use ListItem for actions that navigate away — pair with a router `Link` wrapper at the consuming layer instead
