# AvatarLabel

Combines a Tonic `Avatar` with a title and an optional subtitle. Use it wherever a user, account, or entity needs a visual identity badge alongside readable metadata - profile headers, list rows, and table cells.

---

## Usage

```tsx
import { AvatarLabel, StarIcon } from '@bmi/mui-tonic-components';

<AvatarLabel
  title="John Doe"
  subtitle="Account Manager"
  avatarIcon={<StarIcon />}
/>
```

## Accessibility

- The avatar is purely decorative when it contains an icon - ensure the `title` prop always provides the accessible label for the entity.
- When using an image inside the avatar via `AvatarProps`, pass a descriptive `alt` string through `AvatarProps.alt`.

---

## Do / Don't

**Do**
- Keep titles concise - one or two words for names, short entity identifiers for codes
- Use `subtitle` to surface the primary attribute of the entity (role, status, account number)
- Use `subTitleSuffix` for badge-style inline metadata that must stay on the same line as the subtitle

**Don't**
- Don't pass multi-line strings as `subtitle` - use the `subTitleSuffix` pattern or compose a custom `title` node instead
- Don't rely on `subtitleColor` for semantic meaning (error, success) without also providing a text label
