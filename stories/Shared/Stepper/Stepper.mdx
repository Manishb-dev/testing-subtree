# Stepper

Horizontal step indicator that visualises progress through a multi-step flow. Supports sub-steps within major steps, compact/mobile mode, and optional click-to-navigate.

---

## Usage

```tsx
import { Stepper } from '@bmi/mui-tonic-components';
import type { TonicStepConfig } from '@bmi/mui-tonic-components';

const steps: TonicStepConfig[] = [
  { label: 'Business', subSteps: 3 },
  { label: 'Terms' },
  { label: 'Payment' },
];

<Stepper
  steps={steps}
  activeStep={2}
  lastCompletedStep={1}
/>
```

## Under the Hood

This is a **shared** component that renders a custom flex layout with numbered step labels and progress bars matching the Figma spec.

**Theme override:** `packages/theme/src/overrides/stepper.ts`

The `generateRenderItems` helper transforms the step config into an alternating list of labels and bar groups. Steps with sub-steps consume one flat index per sub-step; steps without sub-steps consume one flat index each.

## Accessibility

- Each step uses `MuiStepButton` which renders as a `<button>` with `role="tab"` and `aria-selected` based on active state.
- Completed steps are keyboard-navigable when `onStepClick` is provided; disabled steps are skipped in tab order.
- Screen readers announce the step label and its completion state via MUI's built-in ARIA attributes.
- Sub-step connectors are decorative — they use `MuiStepConnector` as a visual-only icon and do not receive tab focus when `onStepClick` is omitted.

---

## Do / Don't

**Do**
- Use flat indices consistently — the consumer owns the mapping between logical step and flat index
- Provide `lastCompletedStep` to control progress explicitly (pass `activeStep - 1` for standard linear flows)
- Use `compact` on viewports under ~400px for a mobile-friendly layout

**Don't**
- Don't put business logic inside the stepper — it's a dumb visual indicator
- Don't rely on internal state — pass `activeStep` and optionally `lastCompletedStep` from the consumer
- Don't use `onStepClick` for steps that aren't completed — the component disables them automatically
