# AmountEditField

Inline-editable currency amount field that toggles between a formatted read-only display and a text input on click. Used for payment summaries, invoice amounts, and anywhere users need to quickly adjust a monetary value.

---

## Usage

```tsx
import { AmountEditField } from '@bmi/mui-tonic-components';

const [inputValue, setInputValue] = useState('1750');
const [displayValue, setDisplayValue] = useState('1,750');

<AmountEditField
  displayValue={displayValue}
  inputValue={inputValue}
  onInputChange={setInputValue}
  onCommit={(v) => setDisplayValue(formatCurrency(v))}
  prefix="$"
  size="small"
/>
```

## Under the Hood

This is a **shared** component composed from MUI primitives (`TextField`, `InputAdornment`) and Tonic base atoms (`Box`, `Stack`, `Typography`). It also uses `PencilSimpleIcon` from `@bmi/mui-tonic-icons`.

**MUI base:** `@mui/material/TextField` (edit mode), `@bmi/mui-tonic-components/Typography` (display mode)

The component manages two pieces of UI-only state internally:
- `isEditing` — toggles between display and input mode
- `isHovered` — drives the pencil icon reveal animation and border appearance

All data, formatting, and validation are the consumer's responsibility.

## Accessibility

- The display mode renders as a clickable `Stack` — screen readers announce it as interactive content. Consider wrapping with `role="button"` and `tabIndex={0}` in the consuming application if keyboard-only activation is required beyond click.
- In edit mode, the `TextField` receives `autoFocus` so keyboard users land directly in the input.
- Enter commits, Escape cancels — standard keyboard patterns for inline editing.
- The `error` prop sets `aria-invalid` on the underlying MUI `TextField` automatically.

---

## Do / Don't

**Do**
- Pre-format `displayValue` with proper locale-aware currency formatting in the consumer
- Use `onCommit` to validate the amount before persisting (e.g. check max/min bounds)
- Use `size="small"` on mobile breakpoints and `size="medium"` on desktop

**Don't**
- Don't put raw numeric values in `displayValue` — always format them first
- Don't rely on this component for validation — validate in `onCommit` in the consumer
- Don't use this component for non-numeric input — it's optimized for currency amounts with `inputMode="decimal"`
