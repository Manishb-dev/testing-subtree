# How to use the packages

A step-by-step guide to consuming `@bmi/tonic-components-mui` in your React application.

---

## 1. Install peer dependencies

Tonic requires MUI v7, Emotion, and the Tonic theme & icons packages as peer dependencies.
Install everything in one go:

<Source language="bash" code={`npm install @bmi/tonic-components-mui @bmi/tonic-theme-mui @bmi/tonic-icons-web \
  @mui/material @emotion/react @emotion/styled`} />

---

## 2. Apply the Tonic theme

Wrap your application root with MUI's `ThemeProvider` and pass in the Tonic theme.
This ensures every Tonic component picks up the correct colours, typography and spacing.

<Source language="tsx" code={`// src/App.tsx
import { ThemeProvider, CssBaseline } from '@mui/material';
import { lightTheme } from '@bmi/tonic-theme-mui';

export default function App() {
  return (
    <ThemeProvider theme={lightTheme}>
      <CssBaseline />
      {/* your application routes / pages */}
    </ThemeProvider>
  );
}`} />

> **Tip:** Tonic also exports a `darkTheme` if your app supports dark mode.

---

## 3. Use components

Import any component from `@bmi/tonic-components-mui` and use it like a regular React component.
All props are fully typed.

---

### Example — component

<Source language="tsx" code={`import { Button } from '@bmi/tonic-components-mui';

export function MyPage() {
  return (
    <Button variant="contained" onClick={() => console.log('clicked')}>
      Pay Now
    </Button>
  );
}`} />

---

### Example — icon

Icons live in their own package. Import them from `@bmi/tonic-icons-web`:

<Source language="tsx" code={`import { StarIcon } from '@bmi/tonic-icons-web';

export function MyPage() {
  return <StarIcon />;
}`} />

---

## Key Rules

1. **Always import components from `@bmi/tonic-components-mui`** - never directly from `@mui/material`.
2. **Never add your own `ThemeProvider` inside a page or component** - the root provider is sufficient.
3. **Icons must be imported from `@bmi/tonic-icons-web`** directly — they are not re-exported from `@bmi/tonic-components-mui`. Make sure `@bmi/tonic-icons-web` is installed in your application.
4. **All visual styling lives in the theme**. If you need a style change, update `@bmi/tonic-theme-mui` - do not add inline overrides.
