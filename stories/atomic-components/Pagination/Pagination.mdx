# Pagination

Navigate between pages of content. Use pagination for tables, lists, or any dataset split across multiple pages. Supports first/last page shortcuts and an optional items-range label.

---

## Usage

```tsx
import { Pagination } from '@bmi/mui-tonic-components';

<Pagination count={20} page={1} label="Items" totalItems={200} rowsPerPage={10} />
```

## Under the Hood

This component wraps [`MuiPagination`](https://mui.com/material-ui/api/pagination/) and renders [`MuiPaginationItem`](https://mui.com/material-ui/api/pagination-item/) internally.

**MUI base:** `@mui/material/Pagination` + `@mui/material/PaginationItem`
**Styles:** `packages/components/src/atoms/Pagination/Pagination.styles.ts`

Navigation icons (first, previous, next, last) are replaced with Tonic Caret icons from `@bmi/mui-tonic-icons`. First and last page buttons are shown by default.

## Accessibility

- The component renders inside a `<nav>` element with `aria-label="pagination navigation"` by default. Override with the `aria-label` prop for context-specific labelling (e.g., `"Results pagination"`).
- Each page button includes the page number in its accessible name.
- First, previous, next, and last buttons have meaningful default aria-labels provided by MUI.

---

## Do / Don't

**Do**
- Use `label` to give users context about where they are in a dataset (e.g., "Items 13-22 of 200")
- Control `page` and `onChange` together for predictable navigation
- Pair with a table or list component — pagination should always accompany paginated content

**Don't**
- Don't use pagination for single-page content or fewer than two pages
- Don't hide first/last buttons unless the dataset is very small — they help users jump quickly
- Don't hardcode page counts — compute `count` from your data (`Math.ceil(total / pageSize)`)
