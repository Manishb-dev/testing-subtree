# Alert

A contextual banner for surfacing feedback messages - warnings, errors, and success confirmations. Use it to communicate system or workflow status to the user.

---

## Usage

```tsx
import { Alert } from '@bmi/mui-tonic-components';

<Alert
  severity="warning"
  title="License Renew Reminder"
  description="Your license is expiring soon. Please renew to avoid disruption."
/>
```

## Under the Hood

This component wraps [`MuiAlert`](https://mui.com/material-ui/api/alert/) and [`MuiAlertTitle`](https://mui.com/material-ui/api/alert-title/).

**MUI base:** `@mui/material/Alert`
**Theme override:** `packages/theme/src/overrides/alert.ts`

All visual styling - background color, left accent bar, icon container, and typography weight - lives in the theme override.

## Accessibility

- The underlying `MuiAlert` renders with `role="alert"` by default, which announces the message to screen readers immediately.
- The `severity` prop maps to `aria-live` attributes managed by MUI - no additional ARIA attributes are needed.
- Ensure `title` is descriptive enough to convey the alert intent without relying on the `description` alone.

---

## Do / Don't

**Do**
- Use `warning` for expiring resources or time-sensitive actions that don't block the user yet
- Use `error` for suspended accounts, failed operations, or access restrictions
- Use `success` to confirm a completed action such as a license renewal or form submission
- Keep `title` to a single short phrase (3–6 words)

**Don't**
- Don't use Alert for persistent navigation or global page headers - use AppBar or Banners instead
- Don't stack more than two Alerts on screen at once; prioritize the most critical
- Don't use `info` severity - it is not part of the Tonic vocabulary for this component
