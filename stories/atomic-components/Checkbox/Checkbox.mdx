# Checkbox

A Tonic wrapper around [MUI Checkbox](https://mui.com/material-ui/react-checkbox/) that enforces the Tonic size vocabulary and provides integrated label support via `FormControlLabel`.

---

## Usage

```tsx
import { Checkbox } from '@bmi/mui-tonic-components';

<Checkbox color="primary" size="medium" label="Accept terms" />
```

## Under the Hood

This component is a thin wrapper around [`MuiCheckbox`](https://mui.com/material-ui/api/checkbox/).

**MUI base:** `@mui/material/Checkbox`  
**Tonic override file:** `packages/theme/src/overrides/checkbox.ts`

All visual styling - sizes, colors, border-radius, padding - lives in the theme override, not in the component. The component’s only job is to enforce the Tonic prop vocabulary and forward everything else to MUI.

## Accessibility

- Always pair a checkbox with a visible label - use the `label` prop or a sibling `<label>` element.
- When used without a label (e.g. in a table row), provide `inputProps={{ 'aria-label': '…' }}` to describe the action.
- The `indeterminate` state must be accompanied by a visible parent-selection affordance so its meaning is clear to all users.
- Disabled checkboxes retain their role in the accessibility tree but are non-interactive - do not rely on CSS alone to communicate disabled state.
- All Tonic theme colors meet WCAG AA contrast requirements against a white background at the standard checkbox sizes.

---

## Do / Don't

**Do**
- Use `color="error"` to indicate a validation failure on a required checkbox (e.g. consent checkbox not checked on form submit).
- Use `size="small"` inside dense tables or filter panels where vertical space is constrained.
- Use `indeterminate` on a parent checkbox that controls a partially-selected group of children.

**Don't**
- Don't use a standalone unchecked checkbox as a primary call-to-action - use a `Button` instead.
- Don't control the `checked` prop without also wiring `onChange` - a controlled checkbox with no handler locks the UI.
- Don't rely solely on color to convey state; ensure disabled and error states are communicated structurally (e.g. helper text, `aria-describedby`).
