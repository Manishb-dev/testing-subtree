# Button

The `Button` component is a thin wrapper around [MUI Button](https://mui.com/material-ui/react-button/), inheriting all its props and behaviour while being themed via the Tonic design system.

---

## Usage

```tsx
import { Button } from '@bmi/mui-tonic-components';

<Button variant="contained" color="primary">Pay Now</Button>
```

## Under the Hood

This component is a thin wrapper around [`MuiButton`](https://mui.com/material-ui/api/button/).

**MUI base:** `@mui/material/Button`  
**Tonic override file:** `packages/theme/src/overrides/button.ts`

All visual styling - border-radius, padding per size, text transform - lives in the theme override. The component additionally handles the `loadingState` prop to render the `DotsLoader` or `CheckCircleIcon` and conditionally apply loading/success `sx`.

## Accessibility

- Always provide meaningful `children` text - avoid icon-only buttons without `aria-label`.
- The `loadingState="loading"` state sets `disabled` internally; screen readers will announce the button as disabled.
- Ensure sufficient color contrast - all Tonic theme colors meet WCAG AA at the button's font size.
- Use `contained` sparingly - only one high-emphasis action should appear per section.

---

## Do / Don't

**Do**
- Use `contained` for the single primary action per view (e.g. Submit, Save, Pay Now).
- Use `outlined` for secondary actions that sit alongside a primary button.
- Use `text` for low-priority or inline actions (e.g. Cancel, Learn more).
- Use `loadingState` to give feedback during async operations without adding extra state UI.

**Don't**
- Don't use more than one `contained` button per section.
- Don't use `text` buttons as standalone CTAs without surrounding context.
- Don't leave `loadingState="loading"` indefinitely - always resolve to `success` or reset.
- Don't hardcode disabled colors or loading styles - let `loadingState` handle it.
