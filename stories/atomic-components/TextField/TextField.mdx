# TextField

Outlined text input for forms, search fields, and data entry. Supports text, password, number, email, select, and multiline modes.

---

## Usage

```tsx
import { TextField } from '@bmi/mui-tonic-components';

<TextField label="Name" placeholder="Enter your name" required />
```

## Select Type

Use the `select` prop with `MenuItem` children to render a dropdown.
The existing Tonic `MenuItem` component is used for option rendering and styling.

```tsx
import { TextField, MenuItem } from '@bmi/mui-tonic-components';

<TextField select label="Category" required>
  <MenuItem value="a">Option A</MenuItem>
  <MenuItem value="b">Option B</MenuItem>
</TextField>
```

## Under the Hood

This component wraps [`MuiTextField`](https://mui.com/material-ui/api/text-field/).

**MUI base:** `@mui/material/TextField`
**Variant:** Always `outlined` — locked by the Tonic wrapper

The label is positioned statically above the input (never floats).
All visual styling lives in the component's styled wrapper.

## Accessibility

- Always provide a `label` — the label is linked to the input via MUI's internal `htmlFor` mechanism.
- Use `helperText` with `error` to surface validation messages to screen readers.
- For select type, keyboard navigation (arrow keys, Enter) works natively through MUI's Select.
- The `required` prop adds both a visible asterisk and the `aria-required` attribute.

---

## Do / Don't

**Do**
- Use `required` to indicate mandatory fields — never add `*` manually to the label.
- Use `helperText` for permanent guidance and for error messaging.
- Use `size="small"` in dense layouts like table rows or sidebars.
- Pass `MenuItem` children when using `select` mode.

**Don't**
- Don't use `placeholder` as a substitute for `label`.
- Don't use `select` for more than ~10 options — consider an autocomplete pattern instead.
- Don't set both `multiline` and `select` — they are mutually exclusive.
- Don't override `variant` — Tonic always uses `outlined`.
