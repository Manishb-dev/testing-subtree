# Foundations Overview

Foundations are the raw design decisions that every component in Tonic is built upon.
They live in the `@bmi/tonic-theme-mui` package and are the **single source of truth** for
visual style - components never hard-code a colour, size, or shadow value; they always
consume a token.

---

## What are Foundations?

Foundations are the lowest-level, non-component design values in the system.
They have no React implementation of their own - they are plain JavaScript constants
that feed directly into the MUI theme and, through it, into every component.

Changing a foundation value in one place updates every component that references it
automatically, in both light and dark mode.

## Token Philosophy

Tonic uses a **two-tier token model**:

### Tier 1 - Raw scales

Numeric colour stops (`blue[500]`), unitless spacing multipliers, and named shadow levels.
These describe *what the value is* - they have no semantic meaning on their own.

```ts
import { blue, Neutral } from '@bmi/tonic-theme-mui';

blue['500']    // #1c81e6
Neutral['900'] // #1d2d38
```

### Tier 2 - Semantic tokens

MUI palette slots (`palette.primary.main`, `palette.text.secondary`) that map a raw scale
value to a *role* in the UI. Semantic tokens are what components consume.

```ts
// Inside a component - reads the semantic token, not the raw scale
sx={{ color: 'primary.main' }}
```

Reason: swapping a theme (e.g. dark mode, brand variant) only requires changing
the Tier 2 mapping. Components remain untouched.

---

## Token Hierarchy

```
Raw scale value          →  Semantic token            →  Component prop
────────────────────────────────────────────────────────────────────────
blue[500] (#1c81e6)      →  palette.primary.main      →  color="primary"
Neutral[900] (#1d2d38)   →  palette.text.primary      →  (default text)
orange[400]              →  palette.warning.main      →  color="warning"
```

The hierarchy ensures that:

1. **Designers** work with semantic names in Figma (e.g. *Primary / Main*).
2. **Developers** consume the same semantic names via MUI sx / styled.
3. **Raw scales** are only referenced in theme configuration files - never in component code.

      <td>Dark backgrounds, light text.</td>
    </tr>
  </tbody>

Both themes share the same raw colour scales; only the semantic mapping differs.
The active theme is set once at the app root via `ThemeProvider` and propagates
automatically to every component - including Storybook stories.

```tsx
import { lightTheme, darkTheme } from '@bmi/tonic-theme-mui';
import { ThemeProvider } from '@mui/material/styles';

<ThemeProvider theme={lightTheme}>
  <App />
</ThemeProvider>
```

> In Storybook, the theme is toggled via the toolbar. No story should import or apply a
> `ThemeProvider` directly.

## Usage Guidelines

- **Always consume semantic tokens** from the palette - not raw scale values - inside components.
- **Never hard-code hex values** in component code or stories.
- **Spacing** must use `theme.spacing(n)` or the `sx` shorthand (`p: 2`, `mt: 1`) - never raw pixel strings.
- **Typography variants** must use an existing variant key (`body1`, `caption`, `h4` …) - do not create ad-hoc font styles.
- **Icons** are imported individually from `@bmi/tonic-icons-web` for tree-shaking.
- **Border radius** is applied via component-level `size` props or `sx={{ borderRadius: n }}` - the base unit comes from `theme.shape.borderRadius`.

## Code Integration

Install the theme and component packages:

```bash
npm install @bmi/tonic-theme-mui @bmi/tonic-components-mui @bmi/tonic-icons-web
```

Set up the theme provider in your application root:

```tsx
import { lightTheme } from '@bmi/tonic-theme-mui';
import { ThemeProvider, CssBaseline } from '@mui/material';

export function App() {
  return (
    <ThemeProvider theme={lightTheme}>
      <CssBaseline />
      {/* your app */}
    </ThemeProvider>
  );
}
```

Consume foundation tokens inside components via MUI's `sx` prop or `styled`:

```tsx
// ✅ Correct - uses semantic tokens
<Box sx={{ color: 'text.secondary', p: 2, borderRadius: 1 }} />

// ❌ Wrong - hard-coded values bypass the token system
<Box sx={{ color: '#73838f', padding: '16px', borderRadius: '4px' }} />
```

Reference a raw scale only when configuring the theme itself:

```ts
// theme file only
import { blue, Neutral } from '@bmi/tonic-theme-mui';

primary: {
  main: blue['500'],
  dark: blue['600'],
  contrastText: Neutral['000'],
}
```
