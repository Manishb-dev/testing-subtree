# Contribution Guide

This document explains how to contribute to Tonic — BMI's internal design system.
Follow these rules exactly to keep the codebase consistent and maintainable.

## Repository Structure

```
bmi-tonic/
  packages/
    components-core/     # @bmi/tonic-components
      src/
        atoms.ts         -- re-exports shared atom type contracts
        shared.ts        -- re-exports shared component type contracts
        index.ts         -- barrel export
    components-mui/      # @bmi/tonic-components-mui
      src/
        atoms/           -- typed MUI wrappers (Button, Select, Checkbox…)
        shared/          -- composed reusable components (AvatarLabel, Stepper…)
        index.ts         -- barrel export
    components-tamagui/  # @bmi/tonic-components-tamagui
      src/
        atoms/           -- Tamagui atom wrappers (Button…)
        index.ts         -- barrel export
    icons-web/           # @bmi/tonic-icons-web
      src/               -- one file per icon + index.ts (web / MUI)
    icons-native/        # @bmi/tonic-icons-native
      src/               -- one file per icon + index.ts (React Native)
    theme-core/          # @bmi/tonic-theme
      src/               -- shared design tokens
    theme-mui/           # @bmi/tonic-theme-mui
      src/               -- MUI theme config built from core tokens
    theme-tamagui/       # @bmi/tonic-theme-tamagui
      src/               -- Tamagui config built from core tokens
    storybook/           # documentation site (this app)
      .storybook/
        main.ts          -- Storybook config, vite aliases, addons
        preview.ts       -- global decorators, globalTypes (theme / locale / framework)
        manager.ts       -- sidebar filter logic (framework switcher)
      stories/
        MUI/             -- MUI component stories
        Tamagui/         -- Tamagui component stories
        Foundations/     -- Design tokens, colours, spacing, typography
        Icons/           -- Icon catalogue
```

---

## 1. Adding a Component

### Step 1 — Choose the target framework

| Framework | Package | When to use |
|-----------|---------|-------------|
| MUI | `@bmi/tonic-components-mui` | Web-only applications using MUI |
| Tamagui | `@bmi/tonic-components-tamagui` | Cross-platform (web + React Native) |

### Step 2 — Classify the component layer

<table>
  <thead>
    <tr>
      <th>Layer</th>
      <th>When to use</th>
      <th>Location</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><strong>Atoms</strong></td>
      <td>Typed wrapper around a single framework component. No composition, no state.</td>
      <td><code>atoms/[ComponentName]/</code></td>
    </tr>
    <tr>
      <td><strong>Shared</strong></td>
      <td>Composed from atoms. Reusable across portals. UI-only state allowed.</td>
      <td><code>shared/[ComponentName]/</code></td>
    </tr>
    <tr>
      <td><strong>Patterns</strong></td>
      <td>Full UI section from shared + atomic components. All data via props.</td>
      <td><code>patterns/[ComponentName]/</code> (MUI only for now)</td>
    </tr>
  </tbody>
</table>

If a component imports Redux, i18n, react-router, or any API layer — it does **not** belong in Tonic. It stays in the consuming application.

### Step 3 — Create the component files

Each component lives in its own folder:

```
packages/components-mui/src/[layer]/[ComponentName]/
  [ComponentName].tsx          -- component implementation
  [ComponentName].styles.ts    -- styled() wrappers (MUI only)
  index.ts                     -- re-export
```

### Step 4 — Export from the package barrel

Add an export to the package's `src/index.ts`:

<Source language="ts" code={`export { ComponentName } from './[layer]/ComponentName';
export type { TonicComponentNameProps } from './[layer]/ComponentName';`} />

### Step 5 — Follow naming conventions

- Component files: **PascalCase** (`StatusChip.tsx`)
- Props interface: `Tonic[ComponentName]Props` (always prefix with `Tonic`)
- All props must be typed — no `any`, no untyped objects

## 2. Adding an Icon

Icons live in `packages/icons-web/src/` (web) or `packages/icons-native/src/` (native). Each icon is a separate file.

### Structure

```
packages/icons-web/src/
  [IconName]Icon.tsx    -- icon component
  index.ts             -- barrel re-export
```

### Icon component pattern

<Source language="tsx" code={`import MuiSvgIcon, { SvgIconProps as MuiSvgIconProps } from '@mui/material/SvgIcon';
import Box from '@mui/material/Box';

export type IconSize = 'small' | 'medium' | 'large';
export const sizeMap: Record<IconSize, number> = { small: 16, medium: 24, large: 32 };
export type IconVariant = 'outlined' | 'filled' | 'contained';
export type IconContainedShape = 'circle' | 'square' | 'rounded';

export interface TonicStarIconProps extends Omit<MuiSvgIconProps, 'fontSize'> {
  /** @default 'medium' */
  size?: IconSize;
  /** @default 'outlined' */
  variant?: IconVariant;
  /** @default 'circle' */
  containedShape?: IconContainedShape;
  /** @default 'primary.main' */
  bgColor?: string;
}

export const StarIcon = ({
  size = 'medium',
  variant = 'outlined',
  containedShape = 'circle',
  bgColor = 'primary.main',
  sx,
  ...rest
}: TonicStarIconProps) => {
  // render MuiSvgIcon; if variant='contained', wrap in a Box with background
};`} />

### Rules
- Always use `currentColor` — icon colors must inherit from the parent context
- `viewBox` must match the source SVG exactly
- Export from `packages/icons-web/src/index.ts`
- After adding the icon, add it to `stories/Icons/[IconName]/[IconName]Icon.stories.tsx` and to the Foundations icon catalogue

## 3. Writing Stories

Every component must have a story. Stories live in `packages/storybook/stories/`.

### File location

```
stories/
  MUI/
    Atoms/[ComponentName]/[ComponentName].stories.tsx
    Atoms/[ComponentName]/[ComponentName].mdx
    Shared/[ComponentName]/[ComponentName].stories.tsx
    Shared/[ComponentName]/[ComponentName].mdx
  Tamagui/
    Atoms/[ComponentName]/[ComponentName].stories.tsx
    Atoms/[ComponentName]/[ComponentName].mdx
```

### Title convention

```
MUI atoms:      title: 'MUI/Atoms/[ComponentName]/Tonic[ComponentName]'
MUI shared:     title: 'MUI/Shared/[ComponentName]/Tonic[ComponentName]'
MUI patterns:   title: 'MUI/Patterns/[ComponentName]/Tonic[ComponentName]'
Tamagui atoms:  title: 'Tamagui/Atoms/[ComponentName]/Tonic[ComponentName]'
```

Titles beginning with `MUI/` are shown when the **Framework** toolbar is set to MUI.
Titles beginning with `Tamagui/` are shown when it is set to Tamagui.
Everything else (Foundations, Getting Started, Icons…) is always visible.

### Import rules

- MUI stories: always import components from `@bmi/tonic-components-mui`
- Tamagui stories: always import components from `@bmi/tonic-components-tamagui`
- Always import icons from `@bmi/tonic-icons-web`
- Never import directly from `@mui/material/[Component]` or `tamagui`
- Never add `ThemeProvider` or `TamaguiProvider` inside a story — the global decorator in `preview.ts` handles this

### Do not add stories for this section (`Getting Started/`)

The pages under **Getting Started** are maintained docs pages (MDX only), not interactive stories.
Do **not** add `.stories.tsx` files here.

## 4. Theme Changes

Theme tokens live in `packages/theme-core/src/` (shared) and are extended in `packages/theme-mui/src/` and `packages/theme-tamagui/src/`.

- Never hardcode hex values or font sizes inside a component — always reference a theme token
- If a token does not exist, add it to `theme-core` first, then surface it in both framework-specific themes
- Document new tokens in your PR description

---

## 5. Code Standards

- **TypeScript strict mode** is enabled — all types must be explicit
- **No `any`** — use `unknown` if the type is genuinely unknown and narrow it
- **No direct framework imports in stories** — import from `@bmi/tonic-components-mui` or `@bmi/tonic-components-tamagui`
- **No side effects at module level** — no API calls, no timers, no subscriptions outside React lifecycle
- **Props spread with care** — always explicitly list picked props; avoid spreading unknown objects onto DOM elements

---

## 6. Pull Request Checklist

Before opening a PR, verify:

- [ ] Component is in the correct package (`components-mui` or `components-tamagui`) and correct layer
- [ ] Props interface is named `Tonic[ComponentName]Props`
- [ ] Component is exported from the package's `src/index.ts`
- [ ] A story exists with the correct title prefix (`MUI/…` or `Tamagui/…`)
- [ ] New web icons are added to `packages/icons-web/src/index.ts` and the icon catalogue story
- [ ] No hardcoded colours, spacing, or typography values
- [ ] No `any` types
- [ ] `ThemeProvider` / `TamaguiProvider` is not duplicated inside a story
